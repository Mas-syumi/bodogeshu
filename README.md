# ボドゲ集

ブラウザで遊べる、CPU対戦カードゲーム集です。HTMLファイル1つで完結しています。

**▶ [遊ぶ](./bodoghesyu.html)**

## 収録ゲーム

| ゲーム | 内容 | 人数 |
|---|---|---|
| 🐮 うしのむれ | 104枚の数字カードを4列に並べ、6枚目を置いた人が失点を引き取る押し付け合い | 3〜6人 |
| 💎 宝石商 | 宝石を集めて発展カードを買い、ボーナスを雪だるま式に増やす拡大再生産 | 2〜4人 |
| 💌 恋文 | 手札は常に1枚。捨て札と推理で相手を脱落させる短期決戦 | 2〜4人 |
| 🕵️ 踊る容疑者 | 犯人カードは手から手へ。探偵で暴くか、いぬで嗅ぎ当てるか、逃げ切るか | 3〜8人 |
| 🐺 ジャッカル | 自分のカードだけ見えない。全員の合計を推理して数を釣り上げるチキンレース | 3〜6人 |
| 🪳 むしポーカー | 「これはカエルです」…嘘か本当か。嫌われ者を押し付け合うブラフ勝負 | 2〜6人 |

すべてのゲームに難易度3段階（やさしい / ふつう / つよい）と、遊び方の説明画面、対戦履歴があります。

## CPUについて

難易度「つよい」は、それぞれ本気で作ってあります。

- **うしのむれ** — 未公開カードから相手の手札を推定（決定化サンプリング）し、そこからラウンド終了までを何百回もプレイアウトして、失点の期待値が最小になる札を選びます。完全探索ではなく Perfect Information Monte Carlo による**近似**です。
  ヘッドレスで自動対戦させた実測値は、単純なヒューリスティックCPU相手に平均失点 **7.6 vs 14.1**、勝率 **46.7%**（4人戦の基準値は25%）。
  なお開発中に「1トリック先だけを評価するMC」も比較しており、そちらは平均失点12.8とほぼ改善しませんでした。**ラウンド終了まで回すこと**が強さの本体です。
- **ジャッカル / むしポーカー** — 捨て札や場札から残り枚数を数え、確率を計算します。むしポーカーのCPUは、あなたがあと1枚で負ける生き物を「本当のことを言って」押し付けてきます。
- **踊る容疑者** — 犯人カードの受け渡しを追跡し、隠れた交換が起きるたびに容疑者を広げ、探偵の空振りで絞り込みます。

## 使い方

`bodoghesyu.html` をブラウザで開くだけです。ダウンロードしてローカルで開いても動きます。

### 対戦履歴の同期（任意）

履歴は既定ではブラウザのローカルストレージに保存されるため、端末ごとに別々になります。
Firebase Realtime Database を設定すると、全端末で共有できます。`bodoghesyu.html` 内の `firebaseConfig`（各ゲームに1つずつ、計6箇所）に設定を貼ってください。

```js
const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  databaseURL: "https://xxxx-default-rtdb.firebaseio.com",
  projectId: "...",
};
```

保存パスはゲームごとに分かれています（`nimmt/history`, `gems/history`, `loveletter/history`, `hannin/history`, `jackal/history`, `gokipoker/history`）。

## 技術メモ

- 依存ライブラリなし・ビルド不要。1ファイルにすべて同梱しています
- ランチャーは各ゲームを iframe（srcdoc）で読み込みます
- 個別のゲームファイル（`nimmt.html` など）も単体で動作します
